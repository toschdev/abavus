Core Concepts

      

        
Session

        
A complete interaction period with the agent (from start to end/stop). All actions inside are grouped here.


        
Chronicle / Chain

        
Append-only log of every action. Each entry is cryptographically signed (Ed25519) and linked by hash to the previous one. "Chain valid" means nothing was tampered with.


        
Spool

        
Temporary fast buffer for hook events. Data moves to the permanent signed DB on session end or manual flush. "Spool pending" = not yet in the main log.

      


      
Timeline Item Types

      

        
🧠 llm.turn

        
Full LLM interaction: user messages + model thinking (if any) + final output + any tool calls planned + token usage + cost.


        
🔧 tool.call

        
The agent decided to use a tool (read file, run command, search_replace, web search, etc.). Often includes arguments.


        
💬 message.in

        
User prompt or input that started or continued the interaction.


        
📖 / ✏️ file.*

        
File operations performed by tools (read, write, edit). These are extracted from tool calls for clarity.


        
▶️ / ⏹️ session.*

        
Start or end of an agent session. Marks the boundaries of a conversation or task run.

      


      
Stats Cards

      

        
Entries

        
Total signed log lines in the entire chronicle.

        
Tool calls / LLM turns

        
Count of tool uses vs. full model responses. High tool/turn ratio often means heavy agentic behavior.

        
Spool pending

        
Events waiting to be flushed into the immutable DB.
